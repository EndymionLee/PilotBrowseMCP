# 5. 进化阶段

目标：每次执行都能优化能力实现。

## 进化循环

```
首次访问:          只有浏览器 workflow（学习如何操作）
发现 API 后:       API 成为首选，workflow 保留为兜底
参数优化后:        参数文档化，错误处理完善
网站变更后:        API 失效 -> 兜底到 workflow -> 重新发现
```

## 何时触发进化

| 信号 | 动作 |
|------|------|
| 执行中发现新 API | 作为首选加入能力 |
| API 返回 401/403 | 标记认证为前提条件 |
| 浏览器 workflow 失败 | 标记为过期，重新探索 |
| 发现多个相似 API | 合并为可参数化的能力 |
| 发现 GraphQL 接口 | 用一次查询替代多次 REST 调用 |

每次执行都让能力更健壮、更省 token。
