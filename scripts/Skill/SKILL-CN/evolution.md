# 5. Skill 进化流程

每次任务完成后，回顾执行过程，判断 skill 是否需要改进。

## 1. 识别改进点

查找：

- 缺失的指令
- 低效的步骤
- 反复出现的失败
- 新的成功策略

## 2. 生成补丁

不要重写整个 skill。

生成小补丁：

- 加一条新规则
- 修改现有指令
- 删除过时的引导
- 加一个新示例

## 3. 应用更新

用补丁更新 skill 文件。

示例：

改前：

"能调 API 就调 API。"

改后：

"能调 API 就调 API。对于 SPA 网站，先检查网络请求再解析 DOM。"

## 4. 验证

运行同类任务检查：

- 成功率提高了吗？
- 执行时间变短了吗？
- token 消耗减少了吗？

只有在能提升性能时才保留改动。
